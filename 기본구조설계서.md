# AI 경쟁사 동향 모니터링 웹앱 기본 구조 설계서

---

# 1. 프로젝트 개요

- 과제명: 포스코스틸리온 AI 경쟁사 동향 모니터링 웹앱 개발
- 목적: 전략, 구매, 인사노무 관점의 외부 정보를 자동 수집·분석하여 경영진 및 실무자의 의사결정 속도 향상
- 개발 형태: Flask 기반 Web App
- 개발 환경: Python + Flask + Jinja2 + HTML/CSS/Vanilla JavaScript
- AI/LLM: OpenAI Responses API
- 데이터 수집: 네이버 뉴스 API, 해외 경쟁사 공식 홈페이지, 네이버 증권, 서울외국환중개, 한전, 한국가스공사, LME/TradingEconomics, Open DART, 고용노동부, 국가법령정보센터, 매일경제
- 환경 설정: `.env` 기반 API Key 관리
- 캐시 방식: JSON 파일 기반 로컬 캐시 및 마지막 정상 데이터 fallback
- 유지보수 방향: 기능별 서비스 모듈 분리, 초보자도 흐름을 따라 수정 가능한 구조

# 2. 서비스 구조 개요

```text
사용자(UI)
   ↓
Flask Web Application
   ↓
서비스 로직(Service Layer)
   ├─ 전략 데이터 수집 파이프라인
   │  ├─ 국내/해외 뉴스 수집
   │  ├─ 해외 공식 홈페이지 동향 수집
   │  ├─ 주가/환율/원자재 지표 수집
   │  ├─ 최근 공시 수집
   │  └─ 매일경제 내수 업종 전망 수집
   │
   ├─ 구매 모니터링 데이터 재구성
   │  ├─ 환율
   │  ├─ LME Al/Zn
   │  ├─ 전력/LNG
   │  ├─ 중국 HR(열연)
   │  └─ WTI 유가
   │
   ├─ 인사노무 데이터 수집 파이프라인
   │  ├─ 인사노무 키워드 뉴스 수집
   │  ├─ 고용노동부 공지사항 수집
   │  ├─ 국가법령정보센터 최신법령 수집
   │  └─ 인사노무 AI 브리핑 생성
   │
   ├─ JSON 캐시 저장/조회
   └─ LLM Prompt 생성 및 응답 정규화
   ↓
OpenAI Responses API
   ↓
결과 출력(Web UI)
```

# 4. 기능 구조 설계

## 4-1. 전략 데이터 수집 기능

### 역할

- 네이버 뉴스 API 기반 국내 경쟁사 및 이슈 뉴스 수집
- 해외 경쟁사 공식 홈페이지 우선 수집
- 해외 공식 사이트가 막히거나 오래된 경우 네이버 뉴스 fallback 적용
- OpenAI 기반 해외 뉴스 한국어 번역
- 네이버 증권 기반 주가 수집
- 서울외국환중개 기반 환율 수집
- 한전, 한국가스공사, LME/TradingEconomics 기반 원자재·에너지 지표 수집
- Open DART 기반 최근 7일 공시 수집
- 매일경제 기반 건설·가전·자동차 업종 전망 수집

### 처리 흐름

```text
수집 버튼 클릭
→ SSE 진행률 스트림 시작
→ 뉴스 수집
→ 주가/시장지표/공시/내수 전망 병렬 수집
→ OpenAI Prompt 생성
→ 전략 시사점 5개 생성
→ JSON 캐시 저장
→ 화면 갱신
```

---

## 4-2. 구매 모니터링 기능

### 역할

- 전략 대시보드와 같은 최신 데이터를 구매 관점으로 재배치
- 환율과 원자재 가격을 조달 의사결정에 맞게 강조
- 구매 담당자가 확인해야 할 원가 리스크와 체크포인트 제공

### 주요 판단 요소

| 판단 요소 | 설명 |
|---|---|
| USD/KRW | 수출 채산성 및 수입 원가 판단 |
| LME Al/Zn | 소재 원가 압박 또는 완화 판단 |
| 전력비/LNG | 제조원가 부담 판단 |
| 중국 HR(열연) | F/H(냉연) 제품 가격 선행지표 |
| WTI 유가 | 도료(페인트) 가격 선행지표 |

---

## 4-3. 인사노무 모니터링 기능

### 역할

- 직고용, 노동조합, 근로기준법, 노란봉투법, 중대재해 관련 최신 뉴스 수집
- 고용노동부 공지사항과 국가법령정보센터 최신법령 확인
- 공식 업데이트와 뉴스 흐름을 결합해 노무 리스크 AI 브리핑 생성

### 처리 흐름

```text
인사노무 수집 버튼 클릭
→ SSE 진행률 스트림 시작
→ 인사노무 뉴스 수집
→ 고용노동부 공지사항 확인
→ 국가법령정보센터 최신법령 확인
→ OpenAI Prompt 생성
→ 인사노무 시사점 7개 생성
→ 인사노무 캐시 저장
→ 화면 갱신
```

---

## 4-4. 캐시 및 오류 처리 기능

### 역할

- 외부 API 오류가 있어도 전체 앱이 중단되지 않도록 처리
- 마지막 정상 수집 데이터를 화면에 우선 표시
- API Key 미설정 시 해당 호출을 건너뛰고 화면에 상태 표시

### 캐시 파일

| 캐시 파일 | 용도 |
|---|---|
| `dashboard_cache.json` | 전략/구매 대시보드 최신 정상 데이터 |
| `hr_labor_dashboard_cache.json` | 인사노무 대시보드 최신 정상 데이터 |
| `hr_labor_news_cache.json` | 인사노무 뉴스 수집 결과 |
| `hr_labor_updates_cache.json` | 공식 제도 업데이트 결과 |
| `overseas_translation_cache.json` | 해외 뉴스 번역 결과 |
| `opendart_corp_codes.json` | Open DART 기업코드 목록 |

# 6. Prompt 설계 방향

## 6-1. 전략 Prompt 기본 구조

```text
역할:
당신은 POSCO스틸리온 경영진 보고용 경쟁사 동향 분석가입니다.

입력:
- 국내 경쟁사 뉴스
- 해외 경쟁사 공식/최신 뉴스
- 매일경제 업종 전망
- 환율
- 원자재·에너지 지표
- 최근 7일 공시
- 포스코스틸리온 및 경쟁사 주가

출력:
- 종합 기회
- 내수 동향
- 해외 동향
- 경쟁사 동향
- Risk
```

### 전략 Prompt 설계 원칙

| 항목 | 방향 |
|---|---|
| 관점 | 반드시 POSCO스틸리온 관점으로 해석 |
| 내수 분석 | 건설, 가전, 자동차 업종을 분리 |
| 해외 분석 | 해외 경쟁사 공식 동향을 반드시 반영 |
| 원가 분석 | 전력비, LNG, LME, F/H, 도료 가격을 함께 판단 |
| 환율 분석 | USD/KRW 상승은 수출 채산성에 유리한 신호로 해석 |
| 출력 형식 | `ㅇ섹션|중요도|영향도|헤드라인 : 본문` |
| 문체 | 임원 보고용 종결형 |

# 8. 개발 원칙

| 항목 | 방향 |
|---|---|
| 유지보수 | 기능별 서비스 파일을 분리하여 수정 범위 최소화 |
| UI | PC와 모바일 모두 사용할 수 있는 반응형 대시보드 |
| 신뢰성 | 일부 외부 호출 실패 시 마지막 정상 캐시로 fallback |
| 보안 | API Key와 Secret은 `.env`에서만 관리 |
| 실시간성 | SSE 기반 단계별 진행률 표시 |
| 확장성 | 신규 데이터 소스는 `source_registry.py`와 서비스 모듈 추가로 확장 |
| 보고 품질 | LLM 출력 형식과 문체를 정규화해 임원 보고용으로 표시 |
| 데이터 근거 | 수집 데이터에 없는 사실을 생성하지 않도록 Prompt에서 제한 |
