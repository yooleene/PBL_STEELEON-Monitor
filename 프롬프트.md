# 포스코스틸리온 AI 경쟁사 동향 모니터링 웹앱 개발 프롬프트

## 1. 개발 목표

포스코스틸리온 경영진 및 실무자가 **전략, 구매, 인사노무 관점에서 경쟁사 뉴스, 해외 공식 동향, 주가, 환율, 원자재·에너지 지표, 내수 업종 전망, 최근 공시, 노무·제도 업데이트**를 한 화면에서 확인하고, 수집 데이터를 OpenAI API로 분석해 **경영진 보고용 시사점**을 자동 생성하는 Flask 웹 애플리케이션을 개발한다.

전략 대시보드는 포스코스틸리온 관점의 **5개 시사점**을 생성하고, 인사노무 대시보드는 노무 리스크 관점의 **7개 시사점**을 생성한다. 이 앱은 반복적인 정보 검색 업무를 줄이고, 내수·해외·경쟁사·원가·환율·노무 리스크를 빠르게 판단할 수 있는 **실무형 executive watchboard**여야 한다.

---

## 2. 기술 스택

- 백엔드: Python + Flask
- 템플릿: Jinja2
- 프론트엔드: HTML + Vanilla JavaScript + Custom CSS
- 외부 데이터 수집: requests + BeautifulSoup4
- 뉴스 데이터: 네이버 뉴스 검색 API, 해외 경쟁사 공식 홈페이지, 고용노동부, 국가법령정보센터, 매일경제 업종 기사
- 공시 데이터: Open DART API
- AI 분석: OpenAI Responses API
- 모델: `.env`의 `OPENAI_MODEL`, 기본값 `gpt-5.2`
- 실시간 진행률: Server-Sent Events, `EventSource`
- 캐시 저장: JSON 파일 기반 로컬 캐시, 섹션별 마지막 정상 수집값 fallback
- 반응형: PC + 모바일 지원
- 실행 포트: `5001`

## 5. 핵심 사용자 흐름

1. 사용자가 `http://127.0.0.1:5001`에 접속한다.
2. 첫 화면에서 마지막 정상 수집 캐시 또는 기본 시드 데이터를 먼저 보여준다.
3. 사용자가 `실시간 Data 수집하기!` 버튼을 누른다.
4. `/stream/refresh` SSE 스트림으로 단계별 진행률을 표시한다.
5. 데이터 수집이 완료되면 OpenAI API로 시사점을 생성한다.
6. 최신 결과를 `data/dashboard_cache.json`에 저장한다.
7. 화면을 새로고침해 최신 대시보드를 표시한다.
8. 사용자가 `/hr-labor`에서 `인사노무 Data 수집하기!` 버튼을 누르면 `/stream/hr-labor-refresh` SSE 스트림으로 인사노무 뉴스, 공식 제도 업데이트, AI 브리핑을 별도 갱신한다.
9. 인사노무 최신 결과는 `data/hr_labor_dashboard_cache.json`, `data/hr_labor_news_cache.json`, `data/hr_labor_updates_cache.json`에 나누어 저장한다.

## 8. 데이터 수집 파이프라인

데이터 수집은 `services/pipeline.py`에서 단계별로 실행한다.

```python
STAGES = [
    ("collect_news", "뉴스 데이터를 확인하는 중입니다."),
    ("collect_company", "경쟁사 홈페이지 게시물을 확인하는 중입니다."),
    ("collect_stock", "포스코스틸리온 주가 동향을 가져오는 중입니다."),
    ("collect_market", "환율과 원자재 지표를 정리하는 중입니다."),
    ("collect_filings", "최근 7일 공시를 정리하는 중입니다."),
    ("analyze", "LLM으로 시사점을 생성하는 중입니다."),
]
```

각 단계가 시작될 때와 완료될 때 SSE 이벤트를 전송한다.

```json
{
  "type": "progress",
  "phase": "start 또는 complete",
  "stage": "collect_news",
  "message": "뉴스 데이터를 확인하는 중입니다.",
  "progress": 16,
  "data": {}
}
```

마지막에는 다음 이벤트를 보낸다.

```json
{
  "type": "complete",
  "stage": "done",
  "message": "분석이 완료되었습니다.",
  "progress": 100,
  "data": {}
}
```

주가, 시장지표, 공시, 매일경제 내수 업종 전망은 백그라운드 작업으로 병렬 수집한다. 실제 구현은 `ThreadPoolExecutor(max_workers=3)`를 사용하고, `collect_news` 단계에서 국내 뉴스와 해외 공식 홈페이지 동향을 우선 수집한다.

## 9. 네이버 뉴스 수집 로직

네이버 뉴스 검색 API를 사용한다.

```text
https://openapi.naver.com/v1/search/news.json
```

### 수집 조건

- 키워드별 기본 3건 수집
- `sort=sim` 정확도순을 우선 호출
- 부족할 경우 `sort=date` 날짜순을 추가 호출
- API 요청은 최대 3회 재시도
- 429 응답은 잠시 대기 후 재시도
- 최종 실패 시 사용자 친화적인 오류 메시지 반환

### 수집 필드

```python
{
    "keyword": "포스코스틸리온",
    "title": "HTML 제거된 원제목",
    "description": "HTML 제거된 요약",
    "display_title": "38자 이내 축약 제목",
    "display_description": "표시용 요약",
    "link": "originallink 또는 link",
    "pub_date": "pubDate"
}
```

### 중복 제거 기준

다음 기준 중 하나라도 중복이면 제외한다.

- 정규화한 링크가 동일
- 정규화한 제목이 동일
- 정규화한 설명이 동일
- 제목 + 설명 기반 시그니처가 동일
- `SequenceMatcher` 기반 본문 유사도가 높은 기사
- 토큰 교집합 기준 제목 또는 본문 유사도가 높은 기사
- 같은 키워드에서 같은 발행일이고 내용 토큰이 많이 겹치는 기사

### 해외 경쟁사 뉴스 번역

해외 경쟁사 키워드 뉴스는 공식 홈페이지를 우선 확인하고, 최근 7일 안의 유효한 공식 게시물이 없거나 공식 사이트가 막히면 네이버 뉴스 fallback을 사용한다. 최종 해외 뉴스는 OpenAI API를 사용해 한국어로 번역한다.

- 대상: `Nippon Steel`, `ArcelorMittal`, `BlueScope Steel`, `Baosteel`
- 공식 출처: Nippon Steel News Release, ArcelorMittal Latest News, BlueScope News, Baosteel Company News
- 공식 사이트 결과가 `ready`이고 최근 7일 뉴스가 있으면 `source_type="official_homepage"`로 표시한다.
- 공식 사이트가 오래되었거나 막혔거나 수집 불가이면 네이버 뉴스 fallback 결과를 `source_type="naver_fallback"`으로 표시한다.
- fallback 사유는 `official_stale`, `official_blocked`, `official_unavailable`로 구분한다.
- 출력은 JSON 배열로 파싱한다.
- 번역 결과에 영어가 과도하게 남으면 strict 프롬프트로 1회 재시도한다.
- 번역 실패 시 원문 제목과 요약을 그대로 표시한다.
- 화면에서는 `ENG` / `KOR` 전환 버튼을 제공한다.

### 인사노무 뉴스 수집

인사노무 뉴스는 `services/hr_labor_news.py`에서 전략 뉴스와 별도로 수집한다.

- 키워드: `직고용`, `노동조합`, `근로기준법`, `노란봉투법`, `중대재해`
- 키워드별 최신순 후보를 최대 100건 가져온 뒤 중복을 제거하고 3건씩 표시한다.
- 우선 매체: `mk.co.kr`, `hankyung.com`, `labortoday.co.kr`, `laborlaw.co.kr`
- 링크, 제목, 설명, 본문 유사도 기준으로 키워드 간 중복을 제거한다.
- 결과는 `data/hr_labor_news_cache.json`에 저장하고 30분 TTL 안에서는 캐시를 우선 사용한다.

### 인사노무 공식 업데이트 수집

공식 제도 업데이트는 `services/hr_labor_updates.py`에서 수집한다.

- 고용노동부 공지사항: `https://moel.go.kr/board`
- 국가법령정보센터 최신법령: `https://www.law.go.kr/lsSc.do`
- 검색 키워드: `노란봉투법`, `근로기준법`, `노동조합`, `직고용`
- 법령 검색 결과가 불안정할 때는 주요 법령 상세 링크와 fallback 개정일을 제공한다.
- 결과는 `data/hr_labor_updates_cache.json`에 저장하고 30분 TTL 안에서는 캐시를 우선 사용한다.

## 11. 환율 수집 로직

서울외국환중개 데이터를 사용한다.

```text
http://www.smbs.biz/ExRate/TodayExRate.jsp
http://www.smbs.biz/Flash/TodayExRate_flash.jsp
```

최근 7일 안에서 유효한 환율 데이터 2개 날짜를 찾고, 오늘값과 직전 유효값을 비교한다.

### 수집 및 계산 항목

```text
USD/KRW = 1 USD 기준 원화값
USD/JPY = USD/KRW ÷ JPY/KRW
USD/EUR = USD/KRW ÷ EUR/KRW
USD/CNY = USD/KRW ÷ CNH/KRW
USD/MMK = USD/KRW ÷ MMK/KRW
```

### 표시 방식

- `1 USD = 1,380.00 KRW (+3.20 KRW)`
- 상승은 `+`
- 하락은 `△`
- 보합은 `보합`

## 14. 데이터 캐시 및 오류 처리

전략 대시보드 데이터 캐시는 `data/dashboard_cache.json`에 저장한다. 인사노무 대시보드는 성격이 다른 데이터이므로 별도 JSON 캐시를 사용한다.

```text
data/dashboard_cache.json
data/hr_labor_dashboard_cache.json
data/hr_labor_news_cache.json
data/hr_labor_updates_cache.json
data/overseas_translation_cache.json
data/opendart_corp_codes.json
```

### 기본 원칙

- 앱 최초 접속 시 캐시가 있으면 캐시를 먼저 보여준다.
- 캐시가 없으면 기본 seed 데이터를 보여준다.
- 외부 API 수집에 성공하면 즉시 캐시를 갱신한다.
- 일부 API가 실패해도 전체 앱은 멈추지 않는다.
- 실패한 섹션은 마지막 정상 수집값이 있으면 cached fallback으로 대체한다.
- 마지막 정상 수집 시각을 화면에 표시한다.
- 해외 뉴스 번역 결과는 `overseas_translation_cache.json`에 저장해 동일 기사 재번역을 줄인다.
- 인사노무 뉴스와 공식 업데이트는 각각 30분 TTL을 적용한다.

### 상태값 예시

```python
"data_mode": "seed" | "live" | "cached"
"cache_loaded": True | False
"status_note": "외부 수집 성공 시각 기준 마지막 정상 데이터입니다."
```

### 사용자 친화적 오류 메시지

- 네이버 429: `네이버 뉴스 요청 한도가 잠시 초과되어 마지막 정상 수집값을 유지합니다.`
- 네이버 일반 오류: `네이버 뉴스 호출이 일시적으로 불안정해 마지막 정상 수집값을 유지합니다.`
- OpenAI 연결 오류: `OpenAI 연결이 일시적으로 불안정해 시사점 생성을 건너뛰었습니다.`
- 공시 오류: `공시 데이터 호출이 불안정해 마지막 정상 수집값을 유지합니다.`
- 주가 오류: `주가 데이터 호출이 불안정해 마지막 정상 수집값을 유지합니다.`
- 인사노무 뉴스 오류: `뉴스 호출이 불안정해 마지막 정상 수집값을 유지합니다.`
- 인사노무 공식 업데이트 오류: `공식 사이트 호출이 불안정해 마지막 정상 수집값을 유지합니다.`

## 15. LLM 분석 방식

OpenAI Responses API를 사용한다.

```python
client = OpenAI(api_key=api_key)
response = client.responses.create(
    model=model,
    input=prompt,
)
```

### LLM 역할

LLM은 `POSCO스틸리온 경영진 보고용 경쟁사 동향 분석가`로 동작한다.

### 전략 대시보드 분석 원칙

- 반드시 POSCO스틸리온 관점으로 해석
- 모든 시사점은 이번에 수집된 네이버 뉴스, 해외/국내 경쟁사 뉴스, 해외 공식 홈페이지 뉴스, 매일경제 업종 전망, 환율, 원자재, 공시, 주가 데이터를 근거로 새로 작성
- 내수/해외/경쟁사 동향을 분리
- 내수 동향은 국내 경쟁사 뉴스와 매일경제 주요 업종별 전망을 함께 보고 `[건설]`, `[가전]`, `[자동차]`로 구분
- 경쟁사 뉴스와 매일경제 업종 전망이 상충하면 `혼조` 또는 `엇갈림`으로 명시
- 해외 경쟁사 뉴스는 해외 동향에 반드시 반영
- 공시에서 확인되는 경쟁사 실적/투자/의사결정 신호를 경쟁사 동향과 Risk에 반영
- 중국 HR(열연) 선물 등락은 F/H(냉연) 제품 가격의 선행지표로 해석
- WTI 유가는 도료(페인트) 가격의 선행지표로 해석
- POSCO스틸리온의 핵심 원가 요소는 전력비, LNG, LME(Al/Zn), F/H(냉연), 도료(페인트) 단가로 보고, 환율과 합쳐 수익성 유불리를 판단
- 국내 생산 후 달러로 수출 대금을 회수하므로 USD/KRW 상승은 유리, 하락은 불리하게 해석
- Risk를 감안해 POSCO스틸리온의 구체적 기회를 도출
- 기회와 종합 시사점은 하나의 `종합 기회`로 통합
- 종합 기회 헤드라인은 현황 설명이 아니라 가격 방어, 믹스 조정, 수출 확대, 채널 전환, 원가 대응 같은 실행 지시형 문장으로 작성
- 직접 연결이 약한 정보는 `추정`이라고 표시
- 수집 데이터에 없는 사실은 생성하지 않음
- 각 줄은 1~2문장
- 문체는 `~함.`, `~됨.`, `~보임.`, `~필요함.`, `~해야 함.` 형태의 임원 보고용 종결형
- `~였음.`, `~었음.`, `~하였음.`, `~되었음.` 같은 과거형 종결 금지
- 문장 끝이 `~나`, `~하고`, `~이며`처럼 열려 있지 않도록 완결
- 상세 본문에 `[기타]` 라벨은 사용하지 않음

### 출력 항목

전략 LLM은 반드시 아래 5개 항목을 정확히 생성한다.

```text
종합 기회
내수 동향
해외 동향
경쟁사 동향
Risk
```

### 출력 형식

아래 형식을 반드시 지킨다. 네 번째 칸에는 `한줄 헤드라인`이라는 단어가 아니라 실제 헤드라인 문장을 직접 넣는다.

```text
ㅇ종합 기회|높음/보통/낮음|직접영향/간접영향|실제 실행 헤드라인 문장 : 상세 본문
ㅇ내수 동향|높음/보통/낮음|직접영향/간접영향|실제 내수 헤드라인 문장 : 상세 본문
ㅇ해외 동향|높음/보통/낮음|직접영향/간접영향|실제 해외 헤드라인 문장 : 상세 본문
ㅇ경쟁사 동향|높음/보통/낮음|직접영향/간접영향|실제 경쟁사 헤드라인 문장 : 상세 본문
ㅇRisk|높음/보통/낮음|직접영향/간접영향|실제 리스크 헤드라인 문장 : 상세 본문
```

- `판단됩니다.` → `판단됨.`
- `입니다.` → `임.`
- `합니다.` → `함.`
- 번호, 불릿, 불필요한 서론/결론 제거
- 누락된 섹션은 기본 문구로 채움

## 22. 실행 방법

### 1. 가상환경 생성

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 2. 패키지 설치

```bash
pip install -r requirements.txt
```

### 3. 환경변수 파일 생성

```bash
cp .env.example .env
```

`.env.example`이 없다면 직접 `.env`를 만들고 필요한 키를 입력한다.

### 4. 실행

```bash
flask run --port 5001
```

또는 `app.py` 직접 실행 시에도 동일 포트로 뜨도록 아래처럼 명시할 수 있다. 현재 코드처럼 `app.run(debug=True)`만 두는 경우 직접 실행은 Flask 기본 포트를 사용하므로, 과제 기준 실행은 `flask run --port 5001`을 우선으로 한다.

```python
if __name__ == "__main__":
    app.run(host="127.0.0.1", port=5001, debug=True)
```

브라우저에서 접속한다.

```text
http://127.0.0.1:5001
```
